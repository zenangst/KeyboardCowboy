Groups

Group your workflows to make them contextual.

A group is a container of workflows. Groups can be scoped to only be active when a certain criteria is met, such as having a specific application active and being the front-most application.

To create a new group, simply use the built-in keyboard shortcut ⇧⌘N or use the `Add Group` button in the lower left corner.

New groups are declared as global, to scope them to only apply in a certain context, simply click the elipsis that appears when you hover the newly created group.

Workflows

Have one keystroke fire of your workflow.

A workflow is a composition of a collection of keyboard shortcuts that correspond to one or many commands that will run in sequence.

You can create workflows that contain a target action, like switching or opening applications. Or run entire workflows like opening all your work-related applications as soon as you arrive at work.

Perhaps you are switching between design time and development time, then two separate workflows for jumping between contexts might suite you.

Keyboard Shortcuts

A workflow can be triggered by one or a sequence of shortcuts. It could either be a single key or a key with one or more modifiers.

But we didn't stop there, you can also combine multiple sequences of keyboard shortcuts to use cluster your shortcuts under one leading key.

Commands

Think about commands as plain instruction that macOS should carry out. It can be opening or activating an application, open a file, folder or web page. Or if you want to do something more advanced, you can do this by running Apple- or Shellscripts.   The real power lies inside combining commands together and making them contextual. Context is added on the group level so that you can keep your context tidy and on point.

Core Foundation

We didn’t want to hold back when packing the applications with incredible power, but like Uncle Ben so wisely said; “With great power comes great responsibility”. This led us to decide very early on that we needed a unique development process when building an application that monitors key strokes.

 Keyboard Cowboy should therefor have a transparent development process and be 100% open source, readily available for anyone that wants to dig into the nitty gritty details. 

The motivation behind monitoring rather than registering listeners came from the idea that you could rebind your keys to do what ever you like in any given context. It could be re-mapping ⌥W,⌥A,⌥S,⌥D keys to work as your arrow keys or spark new life into the fn-key by using it as a modifier for your keyboard shortcuts. This simply was not possible with global listeners. 
We also wrote down the following application definition which can be interpreted as our core values:

  • Composable: The user should be able to compose workflows that fit their needs
  • Open: as in open source, it should be distributed freely without any fees or other costs
  • Responsive: it needs to be lighting fast, never blocking the user from achieving the task at hand
  • Easy: to use and customize, with a UI familiar to both novice and expert users
  • Secure: we should never track, log or store anything related to the users I/O
The application is written in Swift and uses the latest SwiftUI additions to render the applications user interface.
